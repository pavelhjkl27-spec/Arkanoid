# Арканоид

Учебный проект по Python: браузерная версия игры «Арканоид» с серверной частью на Flask, базой данных PostgreSQL, авторизацией пользователей, уровнями сложности и таблицей рекордов.

## Описание проекта

Проект представляет собой веб-версию классической игры «Арканоид».

Игрок управляет платформой, отбивает мяч и разрушает блоки. Цель игры — набрать как можно больше очков и пройти уровни, не потеряв все жизни.

В проекте реализована не только сама игра, но и серверная часть: регистрация пользователей, вход в аккаунт, сохранение результатов и таблица рекордов.

## Возможности

* регистрация пользователя;
* вход и выход из аккаунта;
* проверка текущего пользователя;
* сохранение результатов игры;
* просмотр личной истории результатов;
* таблица рекордов;
* несколько игровых уровней;
* разные схемы расположения блоков;
* несколько уровней сложности;
* изменение количества жизней, ширины платформы и скорости мяча в зависимости от сложности;
* клиентская часть на HTML, CSS и JavaScript;
* серверная часть на Flask;
* хранение данных в PostgreSQL.

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/pavelhjkl27-spec/Arkanoid.git
cd Arkanoid
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv
```

### 3. Активировать виртуальное окружение

Для macOS/Linux:

```bash
source .venv/bin/activate
```

Для Windows:

```bash
.venv\Scripts\activate
```

### 4. Установить зависимости

```bash
pip install -r requirements.txt
```

### 5. Настроить переменные окружения

Для работы проекта нужен файл `.env` в корне проекта.

Если в репозитории есть файл `.env.example`, можно скопировать его и переименовать копию в `.env`.

Пример содержимого `.env`:

```env
SECRET_KEY=your-secret-key
DATABASE_URL=postgresql+psycopg2://postgres:your_password@localhost:5432/arkanoid_db
```

Вместо `your_password` нужно указать пароль от своей базы данных PostgreSQL.

Также нужно убедиться, что база данных `arkanoid_db` создана в PostgreSQL.

### 6. Применить миграции базы данных

```bash
flask db upgrade
```

### 7. Запустить серверную часть

```bash
python run.py
```

После запуска сервер будет доступен по адресу:

```text
http://127.0.0.1:5000
```

Проверить работу серверной части можно по адресу:

```text
http://127.0.0.1:5000/api/health
```

### 8. Запустить клиентскую часть

Открой новую вкладку терминала.

Из корня проекта перейди в папку `frontend`:

```bash
cd frontend
```

Запусти локальный сервер:

```bash
python -m http.server 8000
```

После этого открой в браузере:

```text
http://127.0.0.1:8000
```

## Основные маршруты серверной части

### Авторизация

| Метод | Маршрут              | Описание                                    |
| ----- | -------------------- | ------------------------------------------- |
| POST  | `/api/auth/register` | регистрация пользователя                    |
| POST  | `/api/auth/login`    | вход пользователя                           |
| POST  | `/api/auth/logout`   | выход из аккаунта                           |
| GET   | `/api/auth/me`       | получение информации о текущем пользователе |

### Уровни и сложности

| Метод | Маршрут             | Описание                           |
| ----- | ------------------- | ---------------------------------- |
| GET   | `/api/levels`       | получение списка игровых уровней   |
| GET   | `/api/difficulties` | получение списка уровней сложности |

### Результаты

| Метод | Маршрут           | Описание                             |
| ----- | ----------------- | ------------------------------------ |
| POST  | `/api/results`    | сохранение результата игры           |
| GET   | `/api/results/my` | получение личной истории результатов |

### Таблица рекордов

| Метод | Маршрут                            | Описание                                          |
| ----- | ---------------------------------- | ------------------------------------------------- |
| GET   | `/api/leaderboard`                 | получение таблицы рекордов                        |
| GET   | `/api/leaderboard?difficulty=easy` | получение таблицы рекордов по выбранной сложности |
| GET   | `/api/leaderboard?limit=10`        | ограничение количества результатов                |

## Уровни сложности

В игре реализованы три уровня сложности:

| Сложность  | Жизни | Ширина платформы | Начальная скорость мяча |
| ---------- | ----: | ---------------: | ----------------------: |
| Лёгкая     |     5 |              170 |                     4.0 |
| Нормальная |     3 |              140 |                     5.0 |
| Сложная    |     2 |              110 |                     6.0 |

Также на разных сложностях отличаются максимальная скорость мяча, частота ускорения и величина ускорения.

## Игровые уровни

В проекте реализовано несколько уровней с разными схемами блоков:

* «Классика»;
* «Пирамида»;
* «Ромб»;
* «Башни»;
* «Волна».

На уровнях может отличаться не только расположение блоков, но и количество мячей.
